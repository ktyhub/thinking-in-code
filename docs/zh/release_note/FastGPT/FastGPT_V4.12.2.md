# FastGPT V4.12.2
## 为什么要使用 FastGPT

在人工智能席卷全球的今天，每个开发者、每个团队都面临着一个核心矛盾：我们渴望拥有强大而灵活的AI能力，却被高昂的开发成本、复杂的技术栈和漫长的迭代周期所束缚。你是否也曾深夜加班，只为调试一个对话流程？是否也曾羡慕大厂的前沿应用，却苦于没有足够的资源去复现？FastGPT 的出现，正是为了解决这样的痛点和矛盾。它不仅仅是一个工具，更是一种解放——让你从繁琐的底层开发中脱身，专注于创造真正的业务价值。选择 FastGPT，就是选择了一条高效、可控且极具创新的智能应用开发捷径。

## FastGPT是什么

FastGPT 是一个基于大型语言模型（LLM）的开源知识库问答与应用构建平台。它允许开发者通过直观的可视化界面，快速搭建、调试并部署高质量的 AI 应用，如智能客服、企业知识库和自动化工作流，而无需深入底层模型技术细节。其核心优势在于将复杂的 AI 能力封装成易于使用的模块，大幅降低了AI应用的开发现代化门槛。

## 入门示例

想象一下，你正在为一家电商公司开发一个智能客服系统。顾客会询问订单状态、退货政策、产品细节等各种问题。

**传统开发方式**：你需要收集大量问答对，训练一个意图识别模型，再为每个意图编写对应的回答逻辑，整个过程耗时耗力且难以维护。

**使用 FastGPT**：
1.  **准备知识**：将公司的产品手册、退货政策文档上传到 FastGPT 知识库。
2.  **构建应用**：在 FastGPT 的可视化工作流编辑器中，拖拽组件搭建一个流程。例如：`用户提问 -> 知识库检索 -> AI 模型生成回答 -> 回复用户`。
3.  **调试与部署**：在界面上直接输入测试问题（如“如何退货？”），系统会自动从知识库中找到相关条款并由AI生成流畅、准确的回复。满意后，一键通过API或嵌入页面部署到网站。

整个过程，你无需编写模型训练代码，只需关注业务逻辑和知识内容，开发效率提升了数个量级。

## FastGPT V4.12.2版本更新了什么

本次更新主要聚焦于增强可控性、优化用户体验和提升系统稳定性。它允许为不同向量模型独立设置并发请求，解决了部分模型不支持高并发的问题。管理员现在可以在对话页配置精选应用和快捷应用，方便团队协作。工作流模块得到了多项优化与异常检测增强，减少了递归可能带来的问题。此外，还修复了包括独立对话页UI异常、移动端分享逻辑错误在内的多个关键问题。

## 更新日志

### 🚀 新增内容

1.  向量模型并发请求设置，不统一设置成 10，避免部分向量模型不支持并发，默认均为 1，可在模型配置中设置。
2.  对话页支持管理员配置精选应用，便于推荐给团队成员使用。
3.  对话页首页，支持管理员配置快捷应用，可以设置团队常用的应用。
4.  支持关闭团队的对话首页。

### ⚙️ 优化

1.  增加工作流**独立分支**异常检测。
2.  向量模型超过 1536 维度进行截断时，强制进行归一化。其他维度是否归一化，完全由配置决定，减少自动判断的计算量。
3.  模型提供商配置移至 plugin sdk 中。
4.  封装 LLM 调用函数，简化 LLM 请求和工具调用。
5.  优化工作流调度代码，避免深度递归。
6.  工作流递归判断优化，对递归线继续分组检测，适配更多样连线。

### 🐛 修复

1.  修复独立对话页部分 UI 异常。
2.  修复独立对话页无法渲染插件交互的问题。
3.  修复多选选择器导致的页面崩溃。
4.  修复移动端分享链接时，异常加载登录态对话页导航的问题。
5.  修复用户同步可能出现的写冲突问题。
6.  修复无法完全关闭系统套餐，存在空对象默认值导致鉴权异常的问题。
7.  修复工作流中添加团队应用时搜索无效的问题。
8.  修复应用版本 ref 字段错误，导致无法正常使用的问题。

### 🔨 工具更新

1.  修复 Doc2x 工具响应值异常。
