# harbor v2.12.1
### 为什么要使用harbor

在当今快速发展的技术世界中，企业面临着如何高效管理和保护其容器镜像的巨大挑战。传统的镜像仓库往往无法满足安全性、可扩展性和管理便捷性的需求。此时，Harbor作为一个开源的云原生注册中心，恰好填补了这一空白。它不仅提供了强大的安全性和灵活的权限管理，还支持多种存储后端，帮助企业在复杂的环境中保持高效和安全。然而，许多开发者仍在犹豫，是否应该将Harbor纳入他们的技术栈。这个矛盾的存在，正是因为对新技术的未知和对现有解决方案的依赖。使用Harbor，意味着拥抱更高效的工作流程和更安全的镜像管理。

### harbor是什么

Harbor是一个开源的云原生注册中心，旨在帮助用户管理和保护容器镜像。它提供了丰富的功能，包括角色基础的访问控制、镜像复制、漏洞扫描和审计日志等，确保用户能够安全、高效地管理其容器镜像。

### 入门示例

想象一下，一个初创公司正在开发一款基于微服务架构的应用程序。开发团队需要频繁地构建和部署容器镜像，但他们面临着如何安全存储和管理这些镜像的挑战。通过使用Harbor，团队可以轻松地将所有镜像集中存储在一个安全的地方，并利用Harbor的权限管理功能，确保只有授权的开发人员能够访问特定的镜像。此外，Harbor的漏洞扫描功能可以帮助团队在镜像发布之前识别潜在的安全风险，从而提高应用程序的安全性。

### harbor v2.12.1版本更新了什么

Harbor v2.12.1版本更新了多个组件，修复了机器人账户创建问题和CVE导出权限问题，提升了安全性。此外，更新了Trivy到v0.57.1和Trivy适配器到v0.32.1-rc.1，确保用户能够获得最新的漏洞扫描功能。还对Go版本进行了升级，以提高性能和兼容性。
